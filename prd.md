# PRD 标注版｜代理终端移动端群管理工作台

> 本文件由新版《代理终端移动端群管理PRD.docx》按页面模块聚合生成，用于和原型角标一一对应。编号 `[1]` 至 `[18]` 已反向写入每个需求模块起始位置；页面中相同编号的角标浮窗展示同一模块的完整研发说明。

## [1] 需求描述：产品定位与范围边界

### 显示样式
- 产品名称：代理终端移动端群管理工作台。
- 所属系统：租赁业务移动端 / 代理端 / 门店端。
- 版本：V1.0。
- 覆盖终端：移动端 H5 / App 内嵌页面。
- 核心用户：代理、门店店长、店员、主播、课程负责人。

### 业务定义
- 一句话定位：面向代理、门店店长、店员、主播和课程负责人的移动端工具，用于查看和管理门店群、直播群、课程群及对应客户，支撑售前咨询、直播互动、售后跟进和课程内容触达。
- 产品不是 PC 后台，不是总部后台，不是客户登录端，也不是完整 IM 聊天工具。
- 核心职责：查看群、查看客户、同步成员、处理异常、承接直播 / 课程 / 门店客户触达。
- 本期重点不是后台配置，后台已有门店创建、客户归属、直播间配置、课程录播配置、可见门店范围等基础能力；移动端负责把这些后台对象变成一线可操作的群关系入口。

### 本期做
- 首页工作台：展示我的群、下级门店群、直播群、课程群、待处理事项。
- 群管理：查看门店群、直播群、课程群、我的群。
- 群详情：查看群基础信息、群主、成员、客户、来源对象。
- 客户管理：按角色查看门店客户、直播客户、课程客户。
- 成员同步：处理新增客户、移除客户、店员变更、群主异常等。
- 待处理任务：展示待同步、群主异常、客户转移、黑名单客户等。
- 角色权限：根据代理、店长、店员、主播、课程负责人展示不同数据。
- 后续触达入口：预留推商品、推优惠券、推直播、推录播入口。
- 原型补充：根据评审反馈，页面已追加轻量群聊承接能力，支持文字、照片、文件、语音和快捷话术，用于业务触达闭环；这不等同于完整 IM 平台。

### 本期不做
- PC 后台：后台已有。
- 总部移动端登录：总部不作为移动端核心用户。
- 客户登录端：V1.0 客户不登录，只作为群成员和触达对象。
- 完整 IM 聊天：PRD 原始边界不做完整 IM；原型仅补充轻量群聊触达。
- 训练营：课程 / 录播不是训练营。
- 复杂数据报表：移动端只展示轻量数据。
- AI 群运营：后续扩展。
- 商品 / 优惠券真实推送：V1.1 接入，本期预留入口。

### 备注
- 研发需要以“群关系管理 + 成员同步 + 异常处理 + 触达入口”为主线实现，不要将本产品扩展成后台配置系统或客户 C 端。

## [2] 需求描述：角色视角与权限规则

### 显示样式
- 角色切换入口展示在顶部身份区。
- “我的”页面展示组织身份、数据范围和权限清单。
- 权限结果用“是 / 否 / V1.1”表达，便于研发验证按钮是否禁用。

### 角色定义
- 代理：管理一批门店的人。核心诉求是查看自己的群、下级门店群、直播群、课程群。
- 门店店长：门店负责人。核心诉求是管理门店群，查看门店客户，处理客户同步。店长默认是门店群群主。
- 店员：门店员工。核心诉求是查看自己相关客户和所在群。通常不具备高权限操作，只做查看和客户跟进。
- 主播 / 直播负责人：直播群群主。核心诉求是查看直播群、覆盖门店、直播客户。
- 课程负责人：课程 / 录播群群主。核心诉求是查看课程群、录播内容、覆盖客户。
- 客户：群成员和触达对象。V1.0 不登录移动端。

### 数据权限
- 代理：自己的群、下级门店群、下级客户、相关直播群、课程群。
- 店长：本门店群、本门店客户、本门店参与的直播群 / 课程群。
- 店员：自己服务客户、自己所在群。
- 主播：自己负责的直播群及群内客户。
- 课程负责人：自己负责的课程群及群内客户。

### 操作权限
- 查看群：代理、店长、店员、主播、课程负责人都可以。
- 查看客户：代理、店长、店员、主播、课程负责人都可以。
- 查看下级门店群：仅代理可以。
- 同步门店群成员：代理和店长可以，店员 / 主播 / 课程负责人不可以。
- 同步直播群成员：代理和主播可以；店长可查看；店员 / 课程负责人不可以。
- 同步课程群成员：代理和课程负责人可以；店长可查看；店员 / 主播不可以。
- 更换门店群主：代理可以，店长可配置，其他角色不可以。
- 更换直播群主：代理可配置，主播可配置，其他角色不可以。
- 更换课程群主：代理可配置，课程负责人可配置，其他角色不可以。
- 停用群：代理、店长、主播、课程负责人可配置，店员不可以。
- 推送商品 / 推送优惠券：V1.1 接入，本期为禁用或预留入口。

### 异常流程
- 无权限时按钮不应隐藏所有上下文，应展示禁用状态或提示“当前身份暂无权限”。
- 店员遇到高风险任务时只能查看或提交上级处理，不能直接执行同步、群主变更或停用。

## [3] 需求描述：移动端信息架构与底部导航

### 显示样式
- 底部 Tab 固定四项：首页、群管理、客户、我的。
- 页面标题与底部导航联动，高层页面可展示搜索；详情页面展示返回按钮。

### 页面结构
- 首页：当前身份、我的群概览、下级门店群、待处理任务、快捷入口、最近动态。
- 群管理：全部群、我的群、下级门店群、门店群、直播群、课程群、群详情。
- 客户：我的客户、门店客户、直播客户、课程客户、客户详情。
- 我的：个人信息、我的代理、我的门店、我的直播、我的课程、权限说明。

### 交互与排序
- 底部 Tab 切换时保留角色身份，但清空页面搜索条件。
- 详情页返回上一层，若没有历史则回到底部 Tab 对应主页面。
- 首页、群管理、客户支持搜索群 / 客户 / 门店 / 课程。

### 备注
- 底部导航是移动端主骨架，不承载业务规则，只负责入口和状态联动。

## [4] 需求描述：首页工作台与角色化数据概览

### 页面目标
- 用户登录后快速看到当前身份、自己能管理哪些群、有哪些待处理任务、需要关注的客户和群变化、常用操作入口。

### 显示样式
- 顶部：当前身份 + 当前范围 + 搜索。
- 中部：数据概览卡片。
- 中部：快捷入口。
- 中部：同步策略 / 业务闭环 / 待处理任务。
- 底部：最近动态。

### 角色化指标
- 代理首页：我的群、下级门店群、门店客户数、直播群、课程群、待处理。
- 店长首页：门店群、门店客户、店员数、直播群、课程群、待同步。
- 店员首页：我的客户、所在门店群、直播群、课程群、待跟进客户。
- 主播首页：我的直播群、今日直播、覆盖门店、覆盖客户、待处理。
- 课程负责人首页：我的课程群、覆盖门店、覆盖客户、录播内容、待处理。

### 交互与排序
- 指标跟随当前角色切换实时变化。
- 待处理数应排除已完成任务。
- 同步类任务、人工确认任务应在首页有明显聚合提示。

### 备注
- 首页不是报表页，不做复杂数据分析，只展示一线决策所需的轻量数量和下一步动作。

## [5] 需求描述：首页快捷入口

### 显示样式
- 快捷入口以宫格按钮展示，按钮包含图标和短文本。
- 无权限或未接入能力以禁用样式展示，不要误导为可执行。

### 入口定义
- 查看群：进入群管理。
- 查看客户：进入客户列表。
- 待处理：查看待处理任务。
- 同步成员：进入待同步列表或打开同步确认。
- 直播群：快速查看我的直播群。
- 课程群：快速查看我的课程群。
- 下级门店群：代理可用，其他角色禁用或提示无权限。
- 推送商品：V1.1 预留。
- 推送优惠券：V1.1 预留。

### 交互与排序
- 同步成员默认选择当前权限内存在待同步的群。
- 没有同步权限时提示“当前身份暂无该群同步权限”。
- 推送商品 / 优惠券本期只保留入口，不执行真实推送。

## [6] 需求描述：自动识别与分级同步策略

### 显示样式
- 首页展示“自动识别 + 分级同步”策略说明。
- 群详情展示当前群的同步策略、低风险数量、高风险数量。
- 变化明细页展示“自动执行 / 人工确认”分层。

### 业务定义
- 系统自动识别门店、直播、课程范围变化后产生的新增、移除、去重、黑名单、角色变更、群主异常等差异。
- 低风险自动同步：新客户加入、重复客户去重、黑名单不加入。
- 高风险人工确认：客户移出、客户转移、群主变更、群人数过多、离职成员移出、异常成员处理。

### 同步触发场景
- 门店新增客户：新客户需要加入门店群。
- 门店客户移除：客户离店或转移门店。
- 店员离职：店员需要从门店群移除。
- 店长变更：群主需要变更。
- 直播可见门店变化：直播群客户范围变化。
- 课程可见门店变化：课程群客户范围变化。
- 客户黑名单：客户需要从群中移除或被排除。

### 异常流程
- 自动同步只处理低风险变更，并生成可追溯记录。
- 高风险变更必须进入待处理任务或同步确认页，由有权限角色执行。

## [7] 需求描述：待处理任务中心

### 页面目标
- 统一承接群成员变化、客户变化、群主异常、直播 / 课程范围变化等任务。

### 任务类型
- 群成员待同步：群成员与后台来源数据不一致。
- 新客户待加入：新客户属于当前门店或范围。
- 客户待移出：客户不再属于当前门店或范围。
- 群主异常：群主离职、停用或缺失。
- 客户转移：客户从一个门店转移到另一个门店。
- 直播范围变更：直播可见门店变化。
- 课程范围变更：课程可见门店变化。
- 黑名单客户处理：黑名单客户需要移出或排除。
- 群人数过多：群成员超过建议阈值。

### 卡片字段
- 任务类型、关联群、变化摘要、发生时间、查看明细、确认处理。
- 卡片示例：待同步｜五一租赁专场直播群；直播覆盖门店发生变化；新增客户 45 人，移除客户 12 人；生成时间：今天 16:20。

### 交互与排序
- 任务按角色可见范围过滤。
- 低风险自动同步任务可查看自动同步记录。
- 高风险任务点击主按钮进入同步确认或异常处理。
- 无权限角色点击处理时进入“提交上级处理”。

## [8] 需求描述：群管理筛选与群列表

### 页面目标
- 用户查看自己权限范围内的全部群，并按照群类型快速筛选。

### 筛选项
- 代理：全部、我的群、下级门店群、直播群、课程群。
- 店长：我的群、门店群、直播群、课程群。
- 店员：我的群、门店群、直播群、课程群。
- 主播：我的群、直播群。
- 课程负责人：我的群、课程群。

### 群卡片字段
- 群名称。
- 群类型：门店群、直播群、课程群、代理群。
- 来源对象：门店、直播间 / 场次、课程。
- 群主：店长、主播、课程负责人等。
- 客户数。
- 内部成员数。
- 状态：正常、待同步、异常、已结束。

- 群卡片点击进入群详情。
- 群卡片操作支持群详情、进入群聊、查看来源、同步成员。

## [9] 需求描述：群详情基础信息与成员概览

### 页面目标
- 展示群的完整信息，包括来源对象、群主、成员、客户、状态、待处理任务和可执行操作。

### 通用结构
- 群基础信息。
- 群来源信息。
- 成员概览。
- 快捷操作。
- 成员列表。
- 客户列表。
- 同步记录 / 服务记录 / 录播内容。
- 后续触达入口。

### 群基础字段
- 群名称、群类型、群主、管理员、群状态、创建时间、最近同步时间、来源对象。

### 成员概览指标
- 总成员数：群内全部成员。
- 客户数：客户成员数量。
- 内部成员数：店长、店员、主播、课程负责人等。
- 异常成员数：离职、停用、黑名单等。
- 待同步成员数：等待加入或移除的成员。

### 成员 Tab
- 全部、群主、管理员、店员、客户、异常。
- 客户成员卡片展示：客户｜门店｜标签｜加入时间｜状态。
- 内部成员卡片展示：角色｜门店或来源｜加入时间｜状态。
- 异常成员卡片展示：已离职、黑名单、建议移出等处理建议。

## [10] 需求描述：成员同步确认与变化明细

### 页面目标
- 对某个群的成员差异做确认，支持查看明细、按变化类型筛选、勾选执行项。

### 同步确认页
- 标题示例：南山门店客户群成员待同步。
- 本次变化：新增客户、移除客户、新增店员、移除店员、群主变更。
- 操作按钮：查看明细、确认同步。

### 变化明细字段
- 变化类型：新增 / 移除 / 角色变更。
- 成员名称：客户或员工名称。
- 成员类型：客户 / 店员 / 店长 / 主播 / 课程负责人。
- 来源对象：门店 / 直播 / 课程。
- 变化原因：新客户、客户转移、离职、范围变化。
- 建议处理：加入、移除、变更角色。
- 操作：确认、忽略。

### 交互与排序
- 变化明细支持全部、新增、移除、角色变更筛选。
- 每条变化使用开关控制是否纳入本次执行。
- 高风险项使用人工确认标识。
- 确认执行后更新群成员、待同步数、群状态和操作日志。

## [11] 需求描述：来源对象与组织链路

### 业务对象关系
- 代理管理多个门店。
- 门店包含店长、店员和客户。
- 门店群按门店自动生成。
- 直播群按直播间 / 场次可见门店生成。
- 课程群按课程可见门店生成，课程下挂录播。

### 来源对象
- 代理群来源：代理组织 / 代理自身。
- 门店群来源：门店。
- 直播群来源：直播间 / 直播场次。
- 课程群来源：课程 / 录播。

### 显示样式
- “来源对象”弹窗展示来源对象、所属租户、覆盖范围、同步规则。
- “我的组织”展示后台主体、代理、门店、业务群的链路。

### 备注
- 总部只作为后台主体和数据来源，不作为移动端核心用户。
- 来源对象用于解释成员从哪里来，不用于移动端新建后台配置。

## [12] 需求描述：门店群规则与门店群详情

### 生成规则
- 门店创建后，系统自动生成门店群。
- 流程：门店创建 → 系统生成门店群 → 店长 / 店员成为群主或管理员 → 门店客户进入群。

### 群主规则
- 门店有店长：店长为默认群主。
- 门店无店长：指定店员为群主。
- 店长离职：转交新店长或代理处理。
- 店长账号停用：群状态标记异常。

### 成员规则
- 店长：自动加入，默认群主。
- 店员：自动加入，作为管理员或内部成员。
- 门店客户：根据客户归属门店加入。
- 代理：可查看下级门店群，不一定作为群成员。
- 黑名单客户：不加入或提示移出。

### 门店群详情字段
- 门店名称、所属代理、群主、店员数、客户数、群状态、最近同步时间。

### 操作
- 查看门店客户、同步客户、查看店员、更换群主、移除异常成员。
- 推送商品、推送优惠券为 V1.1。

## [13] 需求描述：直播群规则、售前售中售后与服务状态

### 生成规则
- 直播开播或直播场次创建后，根据直播间 / 场次配置的可见门店范围形成直播群。
- 流程：直播间 / 直播场次 → 配置可见门店 → 系统获取门店客户 → 以直播间 / 场次名称创建直播群 → 主播作为群主 → 客户进入直播群。

### 命名规则
- 默认：直播间名称 + 直播群，例如“五一租赁专场直播群”。
- 按场次维度：直播场次名称 + 直播群，例如“5月20日酒水租赁直播群”。

### 成员范围
- 主播：默认群主。
- 助播 / 直播运营：内部成员 / 管理员。
- 可见门店店长：内部成员，可选加入。
- 可见门店店员：内部成员，可选加入。
- 可见门店客户：客户成员。
- 黑名单客户：排除。
- 重复客户：去重。

### 业务用途
- 直播群覆盖完整直播服务链路：售前咨询 → 开播提醒 → 直播中互动 → 商品问题答疑 → 下单问题处理 → 售后问题跟进。

### 直播群状态
- 未开始：直播尚未开始。
- 进行中：直播正在进行。
- 已结束：直播已结束，但群可继续用于售后。
- 待同步：直播可见门店或客户范围变化。
- 已停用：群不再使用。

### 操作
- 查看覆盖门店、查看直播客户、同步客户、查看主播信息。
- 推送开播提醒、推送直播商品为 V1.1。
- 记录售后问题为 V1.2。

## [14] 需求描述：课程 / 录播群规则与录播内容

### 生成规则
- 课程下挂录播内容，课程根据可见门店范围形成课程群。
- 流程：课程创建 → 课程下挂录播内容 → 配置课程可见门店 → 系统获取门店客户 → 生成课程群 → 课程负责人为群主。

### 命名规则
- 课程名称 + 课程群，例如“设备租赁使用课程群”。

### 成员范围
- 课程负责人：默认群主。
- 内容运营 / 讲师：管理员或内部成员。
- 可见门店店长 / 店员：可选加入。
- 可见门店客户：客户成员。
- 黑名单客户：排除。
- 重复客户：去重。

### 与训练营区别
- 课程：支持，作为录播内容的上层对象。
- 录播：支持，挂在课程下。
- 训练营：不支持，不做周期任务、打卡、班级运营。

### 录播内容字段
- 录播标题、录播类型、时长、发布时间、状态。
- 类型示例：商品讲解、使用教程、售后教学。
- 状态：上架 / 下架。

### 操作
- 查看课程客户、查看录播内容、同步客户。
- 推送录播、推送课程通知为 V1.1。

## [15] 需求描述：客户模块与客户详情

### 页面目标
- 不同角色按权限查看客户，并查看客户所在群、所属门店和相关触达记录。

### 客户分类
- 全部客户、我的客户、门店客户、直播客户、课程客户。

### 角色可见客户范围
- 代理：下级门店客户、直播群客户、课程群客户。
- 店长：本门店客户、本门店参与的直播 / 课程客户。
- 店员：自己服务的客户、自己所在群客户。
- 主播：自己直播群内客户。
- 课程负责人：自己课程群内客户。

### 客户列表卡片字段
- 客户姓名 / 昵称。
- 所属门店。
- 服务人。
- 客户标签：高意向、已成交、沉默等。
- 所在群：门店群 / 直播群 / 课程群。
- 最近互动。

### 客户详情字段
- 基础信息：客户姓名、手机号、所属代理、所属门店、服务人、客户标签、客户状态。
- 所在群：群名称、群类型、群主、加入时间、群内状态。
- 触达记录：触达类型、触达内容、触达时间、操作人、状态。

### 备注

## [16] 需求描述：群聊触达能力（文字 / 照片 / 文件 / 语音 / 快捷话术）

### 需求来源
- PRD 原始边界写明 V1.0 不做完整 IM 聊天。
- 原型评审后明确补充轻量聊天能力，用于承接群管理后的售前、售中、售后沟通闭环。

### 显示样式
- 群聊页顶部展示群名称、群类型、来源对象、群主、状态、成员数、客户数、待同步数。
- 直播群聊天页额外展示“直播沟通阶段”：售前、售中、售后。
- 聊天列表支持系统消息、成员消息、我的消息。
- 底部输入区支持文字输入和发送按钮。
- 工具区支持照片、文件、语音。
- 快捷话术横向滚动展示。

### 功能要求
- 文字：输入内容后可发送到当前群。
- 照片：选择图片后生成图片消息，包含图片或占位、文件名或说明。
- 文件：选择文件后生成文件消息，展示文件名和大小。
- 语音：打开语音弹窗后可发送 5 秒或 12 秒语音消息，消息展示波形和时长。
- 快捷话术：点击快捷话术直接发送文本。

### 边界说明
- 不要求完整 IM 的已读未读、撤回、@、长连接、消息漫游、端到端通知和客户登录体系。
- 轻量群聊只是移动端原型里的触达承接能力，研发落地时可按业务消息能力拆分。

## [17] 需求描述：异常处理、群主变更、停用与上级提交

### 群主异常
- 店长离职：提示代理或新店长处理。
- 店员作为群主但离职：提示重新指定群主。
- 主播离职：直播群标记群主异常。
- 课程负责人离职：课程群标记群主异常。
- 群主账号停用：群进入异常状态。

### 客户转移
- 客户从 A 店转到 B 店：A 店群待移出，B 店群待加入。
- 客户所属代理变化：原代理范围移除，新代理范围加入。
- 是否自动处理：V1.0 建议人工确认。

### 黑名单客户
- 黑名单客户待加入：不允许加入。
- 黑名单客户已在群内：提示移出。
- 同步成员时命中黑名单：自动排除或提示排除。

### 群人数过多
- 门店群超过 500 客户提示。
- 直播群超过 1000 客户提示。
- 课程群超过 1000 客户提示。
- 提示文案：当前预计加入客户较多，可能影响群运营效果，建议后续按门店、客户标签或活动拆分群。

### 交互
- 异常处理弹窗展示异常类型、对象、建议操作和处理结果。
- 更换群主弹窗展示当前群主和候选人，确认后回写群主和群状态。
- 停用群弹窗展示当前状态和影响范围，确认后群不可继续用于同步和触达。
- 无权限角色可提交上级处理，任务标记为已提交。

## [18] 需求描述：状态规则、自动建群边界与后续触达入口

### 群状态
- 正常：群正常使用。
- 待同步：成员范围有变化，等待同步。
- 异常：群主异常、成员异常等。
- 已结束：直播或课程已结束，但群保留。
- 已停用：群不可继续使用。
- 同步失败：成员同步失败。

### 成员状态
- 正常：正常群成员。
- 待加入：等待加入群。
- 已移除：已从群中移除。
- 已离职：内部人员离职。
- 黑名单：客户被拉黑。
- 已退群：客户主动退群。
- 停用：账号或客户停用。

### 自动建群边界
- 门店群：门店创建后自动生成，移动端只负责查看、同步和异常处理，不再做“确认创建门店群”。
- 直播群 / 课程群：推荐由后台规则生成，移动端展示并允许同步。
- 可选方案：如果后台能力不足，可做移动端确认创建；不建议手动创建，容易增加一线操作成本。

### 后续扩展
- V1.1 群内业务触达：推送商品、推送优惠券、推送直播提醒、推送录播内容、推送课程通知、售前 / 售中 / 售后问题记录。
- V1.1 推送商品流程：进入群详情 → 点击推送商品 → 选择商品 → 编辑推送文案 → 选择是否附带优惠券 → 预览 → 确认推送 → 记录触达结果。
- V1.1 推送直播提醒流程：进入直播群 → 点击推送开播提醒 → 选择直播场次 → 编辑提醒文案 → 确认推送 → 记录触达结果。
- V1.1 推送录播流程：进入课程群 → 点击推送录播 → 选择课程下录播内容 → 编辑推送文案 → 确认推送 → 记录触达结果。
- AI 能力预留：AI 生成群公告、直播预热文案、商品推荐话术、按客户标签推荐商品、识别高意向客户、总结售前 / 售中 / 售后问题。

### 待确认问题
- 客户是否需要登录移动端：建议 V1.0 客户不登录，只作为群成员和触达对象。
- 直播群按直播间还是按场次生成：产品层统一称“直播群”，底层支持关联直播间或直播场次；长期运营按直播间，活动转化按直播场次。
- 门店群是否完全自动生成：建议 V1.0 自动生成。
- 直播群 / 课程群是否自动生成：推荐后台自动创建，移动端直接查看。

### 最终主链路
- 后台已有代理、门店、客户、直播、课程数据。
- 系统自动生成门店群、直播群、课程群。
- 代理 / 店长 / 主播 / 课程负责人登录移动端。
- 查看自己的群和管理范围内的群。
- 查看客户和成员。
- 处理同步、异常、客户变化。
- 后续承接商品、优惠券、直播、课程触达。
